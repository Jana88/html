Giddy hub, 27 april 2014

Natuurlijk is het niet zo eenvoudig als enkel een repositorie maken en je in het daarvoor bedoelde tekstvak te plakken. Wat heb je verder nog nodig dan? Simpel, je moet een aantal 'basis' html codes kennen om hiermee aan de slag te gaan. Tenzij je natuurlijk met MarkDownPad werkt. Indien dit het geval is, moet je weinig tot geen html codes gebruiken aangezien het programma dat voor jou doet. Wel kan het zijn dat je enige aanpassingen moet doen. Zo heb ik bijvoorbeeld gemerkt dat, indien je de gratis versie van MarkDownPAd gebruikt, je geen afbeeldingen kunt invoeren vanuit je eigen computer. Dit heb ik dan opgelost door gewoon de html code in te voegen op het daarvoor bedoelde stuk. Een andere voorbeeld is de paragraaf archives: blogs 2014 op de website Life's new facts. Ook deze heb ik zelf ingevoerd in een apart MarkDownPad document. Het voordeel als je dit zo doet is dat je de webversie in het rechtervenster kunt zien. Maar, eerst de 'gemakkelijke' html codes.

De meest gebruikte zijn de volgende: html code voor hoofdingen, html code voor paragrafen,, html code voor het invoegen van afbeeldingen, html code voor het invoegen van hyperlinken. Natuurlijk kan het altijd zijn dat de volgende handelingen volgens een andere, of soms makkelijkere, code kan uitgevoerd worden. Ik moet hier eerlijk zijn en toegeven dat mijn kennis over html-coderingen vrij beperkt is. Voor het grootste deel heb ik deze codes gehaald uit tutorials van het internet en door op andere websites te gaan kijken hoe hun codering eruit zag voor bepaalde handelingen (rechter muisklik, elementen inspecteren). Helaas, wegens het feit dat GitHub de codes niet op een goede manier weergeeft, heb ik beslist er een afbeelding van te maken en deze zo te tonen.
